| 组件 | 版本 | 说明 |

## 项目启动

```bash
cd frontend
npm install
npm run dev        # 开发服务器 → http://localhost:5173
npm run build      # 生产构建 → dist/
npm run preview    # 预览生产包
```

开发服务器 `host: 0.0.0.0`，局域网内可访问。API 请求通过 Vite proxy 转发到后端：

```
/api/**  →  http://localhost:8080/api/**
```

后端地址在 `.env.development` 中通过 `VITE_PROXY_TARGET` 配置，默认 `http://localhost:8080`。

## 核心架构设计

### 1. 路由与权限

按角色拆分路由文件（`router/routes/{role}.js`），每条路由通过 `meta.roles` 声明可访问角色：

```js
meta: { roles: ['patient'] }
```

全局守卫 `router.beforeEach` 校验：
- `meta.public` → 放行（登录页/注册/错误页）
- 未登录 → 跳 `/login`
- 角色不匹配 → 跳 `/403`

**开发阶段 BYPASS_AUTH 开关**：`router/index.js` 顶部 `BYPASS_AUTH = true` 时，守卫按访问路径前缀自动注入 mock 登录态（`/patient/*` → 患者、`/doctor/*` → 医生、`/admin/*` → 管理员），无需手动登录即可访问所有页面。上线/联调前改回 `false`。

### 2. 布局策略

| 角色 | 布局 | 组件 |
|:---|:---|:---|
| 患者 | 顶部导航条（C 端风格） | `PatientLayout.vue` + `PatientNavbar.vue` |
| 医生 | 左侧侧边栏（B 端风格） | `DefaultLayout.vue` + `AppSidebar.vue` |
| 管理员 | 左侧侧边栏（B 端风格） | 同上 |

### 3. 接口封装

`api/request.js` 创建 Axios 实例，baseURL `/api`，自动携带 token。响应拦截器处理统一格式 `{ code, message, data }`：

- `code === 200` → 返回 `data`
- `code === 401` → 跳登录
- 其他 → `ElMessage.error(message)` + throw

业务模块 (`api/*.js`) 只封装路径，调用方直接拿 data，不处理响应结构。

### 4. 状态管理

`stores/user.js`（Pinia）持有：
- `token` / `role` / `profile`（姓名/工号等）
- `isLoggedIn`（computed）
- `login()` / `logout()` / `isRole()` 方法
- 持久化到 `localStorage`（key: `hospital:token` / `hospital:user`）

### 5. 样式体系

`styles/tokens.scss` 定义全部 CSS 变量（主色 `#3B82F6`、薄荷绿辅色、Tailwind slate 灰阶、8px 基准间距、阴影/圆角体系），所有组件消费变量、不写裸值。

`styles/element-overrides.scss` 覆盖 Element Plus 默认样式（按钮 36px、表格去竖向边框、输入框描边替代阴影、卡片圆角 10px 等），确保 Element 组件即开即用就符合设计规范。

无额外 CSS 框架。

### 6. 挂号状态中转

`utils/booking.js`：挂号流程中 Schedules → BookConfirm → BookSuccess 三个页面间通过 `sessionStorage` 传递排班对象和预约结果，避免 URL query 过长或后端缺乏 GET `/schedules/:id` 的问题。流程结束后 `clearBooking()` 清理。

## 开发约定

- **组件命名**：PascalCase，文件名与组件名一致
- **路径别名**：`@` → `src/`（vite.config.js 配置）
- **接口字段**：全 camelCase，与后端 JSON 一致
- **日期格式**：展示用 `yyyy-MM-dd`，传输用 ISO 字符串
- **错误处理**：API 层统一拦截，页面只关心成功路径
- **Git 分支**：`feature/frontend-xgy`（你的前端分支）
