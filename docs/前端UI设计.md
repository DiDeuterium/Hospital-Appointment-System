# 医院预约挂号系统 — 前端 UI 重设计方案

> **文档定位**：本方案不修改任何代码，仅作设计指导。后续按本文档逐页落地即可。
> **设计目标**：把当前"能跑但简陋"的演示稿，升级为可作为期末展示作品的、贴近真实医疗 C 端产品（参考挂号网 / 微医 / 京东健康挂号 / 支付宝医疗）的界面。
> **约束**：保持现有技术栈（Vue 3 + Element Plus + Vite），不引入额外 UI 库；后端接口完全沿用 `开发指导文档.md` 中的约定。

---

## 一、当前问题诊断（"丑"在哪）

| # | 问题 | 表现 |
|:---|:---|:---|
| 1 | 风格混乱 | 登录页用蓝→绿渐变（演示风），侧边栏却是 Ant Design 深色 `#001529`，主体又是 Element Plus 默认蓝，三种语言并存 |
| 2 | 缺乏品牌识别 | 顶部 logo 是一个 `+` 字符；色板没体系；字号没层级；阴影/圆角全是 Element 默认值 |
| 3 | 全表格交互 | 患者从科室→医生→排班，三步全是 `el-table`，没有任何"产品感"；号源只用数字"12/30"显示，无视觉冲击 |
| 4 | 信息架构残缺 | 患者无"首页/我的预约提醒"；医生无工作台；管理员无数据看板 |
| 5 | 缺导航上下文 | 整个挂号流程无面包屑、无步骤条，用户容易迷路 |
| 6 | 缺关键页面 | 缺"挂号成功页"、"医生详情页"、"预约详情页"，体验断层 |
| 7 | 空状态/反馈贫弱 | 全部用 `el-empty` 默认；loading 只有遮罩；缺成功/失败的强反馈 |
| 8 | 视觉密度失衡 | 表格 `border` 是 Element 默认细灰边，密集且廉价；卡片缺留白、缺图标 |

## 二、设计基调与参考

### 2.1 设计基调
**"干净、专业、有温度"** —— 医疗系统不能像电商一样花哨，但也不能像后台管理那样冷冰冰。

- **干净**：大面积留白，浅色背景（不是纯白），主信息突出
- **专业**：以"医疗蓝"为主色，辅以"健康绿"作状态色；图形语言克制
- **温度**：在患者端使用插画/图标点缀首页，挂号成功页有正向反馈

### 2.2 参考产品
- 挂号网（guahao.com）：科室列表的卡片设计
- 微医：医生卡片（头像+职称+擅长标签）
- 京东健康：首页快捷入口的栅格布局
- 支付宝-医疗健康：状态色与状态卡片的呈现
- Element Plus Pro 模板：管理员后台的整体结构

## 三、视觉规范（Design Token）

### 3.1 色彩

```
─── 主色（Brand / Primary）── 医疗蓝（理性、可信赖）
    --brand-50    #E6F2FF    背景态、Hover 浅底
    --brand-100   #BFDDFF    禁用、骨架屏
    --brand-200   #80BBFF    辅助分隔
    --brand-500   #1677FF    ★ 主色（按钮、Tab 选中、链接）
    --brand-600   #0958D9    Hover/Active
    --brand-700   #003EB3    Pressed/标题强调

─── 辅助色（Accent）── 健康绿（号源充足、成功、生命体征）
    --accent-50   #E8F8F0
    --accent-500  #0BA774    ★ 状态：剩余号源充足、挂号成功
    --accent-700  #047857

─── 语义色（Semantic）
    --success     #0BA774    （与 accent-500 一致）
    --warning     #F59E0B    号源紧张（剩余 ≤ 30%）
    --danger      #E5484D    号源约满、取消
    --info        #6E7681    辅助文案

─── 中性色（Neutral）── 用一套灰阶替代 Element 默认杂色
    --bg-page     #F5F7FA    页面背景（与现状保持一致）
    --bg-elevated #FFFFFF    卡片/模态背景
    --bg-subtle   #F9FAFB    表格行 zebra、二级容器
    --border      #E5E7EB    分割线、卡片边
    --border-light #F0F1F3   表格行分割
    --text-1      #111827    标题
    --text-2      #374151    正文
    --text-3      #6B7280    次要文案
    --text-4      #9CA3AF    占位、禁用文字
```

**色彩使用约束**：
- 一个屏幕不要超过 3 种语义色组合
- 文本永远不要用纯黑 `#000`，正文用 `--text-2`
- 红色严格保留给"约满 / 取消 / 删除"

### 3.2 字体与字号

```
字体栈：
  -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
  "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif

字号体系（4px 基准）：
  --fs-display   28px / 1.3   首页欢迎语
  --fs-h1        22px / 1.4   页面标题
  --fs-h2        18px / 1.5   卡片标题、模态标题
  --fs-h3        16px / 1.5   小标题、强调字段
  --fs-body      14px / 1.6   ★ 默认正文
  --fs-caption   12px / 1.5   辅助、标签

字重：
  400 正文 / 500 强调字段 / 600 标题
  不使用 700+（避免印刷感）
```

### 3.3 间距与圆角

```
间距（8px 基准）：
  4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48

圆角：
  --radius-sm   4px    标签、Tag
  --radius-md   8px    按钮、输入框、表格行 hover 高亮
  --radius-lg   12px   ★ 卡片、Dialog
  --radius-xl   16px   首页大卡片、空状态插画容器

阴影（淡，避免压迫感）：
  --shadow-sm   0 1px 2px rgba(16,24,40,.05)
  --shadow-md   0 4px 12px rgba(16,24,40,.08)    ★ 卡片 hover
  --shadow-lg   0 12px 32px rgba(16,24,40,.12)   模态、悬浮菜单
```

### 3.4 关键组件风格

#### 按钮
- **Primary**：主色背景、白字、`--radius-md`、`padding 8px 20px`、Hover 加深 1 阶
- **Secondary**：白底、主色边框 1px、主色字
- **Text/Link**：无背景，主色字，Hover 加下划线
- **Danger**：仅用于"取消预约 / 删除"，红色背景
- ❌ 不使用 Element 默认那种"圆滚滚"按钮（高度 32px → 改成 36px，更现代）

#### 卡片（核心组件）
```
┌──────────────────────────────────┐
│ 背景：#FFFFFF                     │
│ 圆角：12px                        │
│ 边框：1px solid #F0F1F3（可选）  │
│ 内边距：20px 24px                │
│ 阴影：默认无；hover 浮起 --shadow-md │
│ Hover 上移：translateY(-2px) 0.2s │
└──────────────────────────────────┘
```

#### 表格
- 替换 `border` 全细线为：仅留行底分隔（`--border-light`），表头有底色 `--bg-subtle`
- 表头字色 `--text-3`、字重 500
- 行 hover 变 `--brand-50` 浅底
- 操作列按钮一律用 text + 主色，不堆叠彩色按钮

#### Tag / 状态标签（取代 Element 默认 type）
| 状态 | 背景 | 文字 | 用途 |
|:---|:---|:---|:---|
| 已预约 | `--brand-50` | `--brand-600` | 进行中 |
| 已完成 | `--accent-50` | `--accent-700` | 成功 |
| 已取消 | `#F3F4F6` | `--text-3` | 中性 |
| 充足 | `--accent-50` | `--accent-700` | 号源 > 30% |
| 紧张 | `#FEF3C7` | `#B45309` | 号源 ≤ 30% |
| 约满 | `#FEE2E2` | `--danger` | 号源 = 0 |

#### 图标
- 全用 Element Plus 自带 Icon（`@element-plus/icons-vue`），不引入新库
- 关键位置图标尺寸 20px，行内 14-16px

## 四、信息架构与导航

### 4.1 总体角色分流

```
                    ┌──────────────┐
                    │  /login      │  统一登录入口（三 Tab 切角色）
                    └──────┬───────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
   /patient/*          /doctor/*           /admin/*
   (患者门户)          (医生工作台)        (管理后台)
```

### 4.2 全局布局策略

**患者端**：顶部导航条 + 内容区（**不要侧边栏**）
> 理由：患者端是 C 端体验，菜单项少（首页 / 我的预约 / 个人中心），用顶部 Tab 更轻盈，类似挂号网 App。

**医生端 / 管理员端**：左侧深色侧边栏 + 顶部条 + 内容区
> 理由：B 端工作台，菜单可能扩展，侧边栏标准做法。

### 4.3 页面清单（完整 17 张页面）

| 角色 | # | 页面 | 路由（建议） | 对应接口 |
|:---|:---|:---|:---|:---|
| 公共 | P0 | 登录 | `/login` | 3 个登录接口 |

| 患者 | P6 | **医生详情** | `/patient/doctors/:docId` | 复用医生信息 + 排班 |

| 患者 | P8 | **挂号确认页**（模态/页面） | `/patient/book/:scheduleId` | `POST /appointments` |

★ 加粗的为相对当前版本**新增 / 重做**的页面。

### 4.4 患者挂号主流程（用户视角）

```
首页 → 科室列表 → 科室下医生列表 → 医生详情(看排班)
                                       │
                                       ▼
                              排班选号(日期+时段卡)
                                       │
                              点击"立即挂号"
                                       │
                                       ▼
                              挂号确认页(信息核对)
                                       │
                                       ▼
                              挂号成功页(预约号 + 须知)
                                       │
                                       ▼
                              [返回首页 / 查看我的预约]
```

每一步都有**面包屑** + **顶部进度条**："选科室 → 选医生 → 选时段 → 确认 → 完成"。

## 八、与后端接口的对齐

本设计**完全不改动后端接口**，所有展示内容都能用现有接口（`docs/开发指导文档.md` §2）拿到。下表说明哪些"看起来很多"的字段实际只是前端拼装：

| 设计中的字段 | 实际来源 |
|:---|:---|
| 医生头像 | 性别 Emoji `M→👨` `F→👩`（无后端字段） |
| 医生擅长方向 Tag | 暂留空或前端 mock（后端表里没有，可作为扩展） |
| 从医年限 | 不显示，或由职称推断 |
| 挂号费 | 表里没有此字段，**写"50 元（系统预留）"或不显示** |
| 就诊时间段 "08:00-11:30" | 前端按 shift 固定值映射（上午=08:00-11:30，下午=14:00-17:00） |
| 科室图标 / Emoji | 前端按科室名关键词映射（"内"→❤、"眼"→👁、"儿"→👶） |
| 科室分类（内科系/外科系） | 前端按名称中含"内/外/儿/眼/耳"分组，纯展示 |
| "本周累计患者数" 等统计 | 前端遍历 listSchedules + appointmentList 聚合计算 |

> 重点：**设计稿不假设新增任何后端字段**。所有"看起来挺丰富"的信息都靠现有接口拼，或加合理 mock 并在 UI 中明示是"系统预留"。
